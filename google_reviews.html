Project Details

  • Project URL: GitHub Link
  • Phase 1: Presentation Link

    • The team built an automated data pipeline to collect and process seafood restaurant reviews from Google Maps and developed an interactive Superset dashboard. Using Apify, reviews from Jan 2022 – Oct 2024 were scraped, stored in Google Cloud Storage (GCS), and processed in MotherDuck DB, with Prefect automating workflows. Data was cleaned by removing unnecessary fields, converting categories to binary, merging translations, and analyzing review images. The dashboard provides insights into review trends, best visit times, and price-rating correlations, aiding both customers and restaurant owners. Future plans include NLP sentiment analysis, ML-driven restaurant insights, and pipeline monitoring.

  • Phase 2: Presentation Link

    • The team built a comprehensive data pipeline and analysis system for restaurant reviews. The system begins with weekly web scraping of Google Reviews data using Apify, followed by an ETL pipeline that processes data through GCP and stores it in MotherDuckDB1. The analysis phase incorporates topic modeling and sentiment analysis, with data preprocessing steps including null value handling and text preprocessing1. The predictive analysis utilizes Non-negative Matrix Factorization for topic extraction and sentiment scoring, culminating in a Light GBM regression model that predicts review ratings based on various features including sentiment scores and categorical data1. The entire pipeline is orchestrated using Prefect, with visualization handled through Superset, and model hyperparameter tuning performed using Weights & Biases1.

  • Phase 3: Presentation Link

    • The team expanded the data pipeline to integrate LLMOps and develop two AI-driven data products: a Restaurant Recommendation Engine for customers and a Text2SQL Market Research Tool for restaurant owners. Using Vertex AI, Pinecone, and Streamlit, the recommendation system provides personalized seafood restaurant suggestions based on user queries, incorporating metadata for contextual accuracy. Meanwhile, the Text2SQL tool allows restaurant owners to conduct market analysis by querying restaurant trends using natural language. Enhancements from Phase 2 include BERT-based sentiment analysis, capturing deeper contextual and implicit sentiment beyond traditional lexicon-based models like VADER. Challenges ahead involve acquiring richer restaurant data (e.g., menus, locations), refining sentiment analysis through advanced NLP techniques, and improving the Streamlit UI for better user experience.
