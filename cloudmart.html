Project Details

  • Project URL: GitHub Link

As part of the MultiCloud, DevOps & AI Challenge, I developed CloudMart, a full-stack e-commerce platform that showcases the integration of cloud-native infrastructure, containerized deployment, serverless computing, and artificial intelligence across multiple cloud providers including AWS, Google Cloud, and Microsoft Azure. The project emphasized building a scalable, automated, and intelligent application using industry-standard DevOps practices, infrastructure-as-code, and AI assistants.

The project began with provisioning AWS resources using Terraform, including S3 buckets, EC2 instances, IAM roles, and DynamoDB tables. A custom virtual workstation was created on EC2 to serve as a development environment. From there, I used Terraform to define and deploy an S3 bucket for storing data and set up DynamoDB tables to serve as the core databases for product listings, orders, and tickets. Serverless Lambda functions were written and deployed to interact with these databases, enabling modular backend functionalities such as listing products and syncing data.

Next, I containerized the frontend (React.js) and backend (Node.js/Express) applications using Docker, and deployed them on Amazon EKS (Elastic Kubernetes Service). Custom Docker images were created and pushed to Amazon ECR, and Kubernetes deployment files were written to handle service creation, replica management, and environment variable injection. LoadBalancer services were used to expose both the frontend and backend to public endpoints, and service accounts were granted IAM roles to securely access other AWS services like DynamoDB and Bedrock.

To automate the build and deployment process, I configured a CI/CD pipeline using AWS CodePipeline and CodeBuild. CodePipeline was connected to a GitHub repository containing the CloudMart application code. The pipeline was designed to automatically build Docker images upon code updates, push them to ECR, and redeploy updated containers to the Kubernetes cluster using kubectl. This enabled seamless delivery of new features and updates with minimal manual effort.

A standout feature of CloudMart was the integration of AI capabilities using both OpenAI and Amazon Bedrock. I configured a Claude 3 Sonnet-powered product recommendation agent through Amazon Bedrock. This agent was connected to the backend via a Lambda function that retrieved real-time product listings from DynamoDB and returned contextual recommendations based on user queries. Additionally, I created a customer support chatbot using OpenAI GPT-4o, configured as an assistant via OpenAI’s platform. This assistant was trained to respond to customer service queries politely and effectively, improving user engagement without needing product-specific access.

To enable cross-cloud analytics, I connected AWS DynamoDB to Google BigQuery. This involved enabling DynamoDB streams, creating a new Lambda function to consume the stream, and using the Google Cloud client libraries to push order data into BigQuery tables in real time. I configured the appropriate GCP service accounts and credentials, ensured secure access control, and defined schemas to allow structured data storage. This setup enabled advanced analytics and dashboarding using BigQuery’s capabilities.

Finally, I implemented sentiment analysis using Azure Text Analytics API. This API was integrated into the backend via environment variables, allowing the platform to evaluate customer reviews and extract insights about product satisfaction. This multi-cloud AI integration showcased how disparate cloud services can be unified into a cohesive, intelligent application architecture.
